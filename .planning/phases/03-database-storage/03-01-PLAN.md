<task type="auto">
  <name>Task 1: Create dictations table migration with RLS policies</name>
  <files>supabase/migrations/001_dictations.sql</files>
  <action>
    Create a SQL migration file that:

    1. Creates `dictations` table with columns:
       - `id` UUID primary key default gen_random_uuid()
       - `user_id` UUID not null references auth.users(id) ON DELETE CASCADE
       - `text` TEXT not null
       - `timestamp` BIGINT not null (Unix timestamp in milliseconds)
       - `duration` INTEGER not null (seconds)
       - `created_at` TIMESTAMPTZ default now()

    2. Enables Row Level Security on the table

    3. Creates RLS policies:
       - "Users can view own dictations": SELECT WHERE user_id = auth.uid()
       - "Users can insert own dictations": INSERT WITH CHECK user_id = auth.uid()
       - "Users can delete own dictations": DELETE USING user_id = auth.uid()
       - No UPDATE policy (dictations are immutable after creation)

    4. Creates index on user_id for query performance

    Do NOT include audio storage columns — transcriptions only per project constraints.
  </action>
  <verify>
    <automated>cat supabase/migrations/001_dictations.sql | grep -E "(CREATE TABLE|ENABLE ROW LEVEL SECURITY|CREATE POLICY|auth.uid)"</automated>
  </verify>
  <done>Migration file exists with table schema, RLS enabled, and all four policies defined</done>
</task>

  <action>
    Create src/lib/dictations.ts with the following exported functions:

    2. `saveDictation(userId: string, text: string, duration: number): Promise<{ data: DictationRow | null; error: Error | null }>`

       - Return inserted row or error

    3. `loadDictations(userId: string): Promise<{ data: DictationRow[] | null; error: Error | null }>`

       - Return array or error

    4. `deleteDictation(userId: string, id: string): Promise<{ error: Error | null }>`

       - Return error if any

    5. `migrateLocalStorage(userId: string): Promise<{ migrated: number; errors: number }>`
       - Read localStorage.getItem('dictations')
       - For each local dictation, check if already exists (by timestamp + text match via Supabase query)
       - Insert non-duplicates
       - Clear localStorage only if all migrations successful
       - Return counts

    Import supabase from './supabase'. Use typed responses. Handle all Supabase errors gracefully.
